# Generating Cantonese Content

Generate validated `.md` content files for Cantonese idioms (歇後語), then verify accuracy and language correctness via parallel subagents.

**Core principle:** Generate content, then spawn two independent verification subagents (fact-check + word usage review) before delivering the final files.

## When to Use

- User provides one or more Cantonese idiom terms (歇後語)
- User asks to create new idiom content files
- User wants to expand the `src/contents/idioms/` collection

## Step 1: Read a Sample of Existing Files

Before generating any content, read **3–5 existing files** from `src/contents/idioms/` to get a feel for tone and format. There is no universal template — each idiom has its own style, so use the sample for general orientation only, not as a strict template to follow.

**You do NOT need to read the schema files.** The `pnpm gen:content` script scaffolds the correct frontmatter structure for you.

## Step 2: Scaffold Files with `pnpm gen:content`

For each idiom term, run:

```bash
pnpm gen:content idioms <term>
```

This creates `src/contents/idioms/<term>.md` with:

- A randomly generated 12-char alphanumeric `id`
- Empty frontmatter fields (`term`, `termJyutping`, `answer`, `answerJyutping`)
- No body content

**Do NOT create files manually.** Always use this command — it handles ID generation, path resolution, and duplicate detection.

## Step 3: Fill In Content

Edit each scaffolded file to populate frontmatter and add body content.

### Frontmatter

Fill in the empty fields generated by the scaffold:

```yaml
---
id: <keep the auto-generated id>
term: <the idiom term>
termJyutping: <Jyutping for term>
answer: <the punchline/answer>
answerJyutping: <Jyutping for answer>
---
```

**Jyutping rules:**

- Lowercase syllables with tone numbers 1-6, space-separated
- Must match regex: `^[a-z]{1,7}[1-6]( [a-z]{1,7}[1-6])*$`
- Example: `daai6 daat3 dei6 maai6 zuk1`
- Verify against known references (粵語審音配詞字庫, words.hk, 漢典)

### Title Format

```markdown
# TERM──ANSWER
```

Use the fullwidth dash `──` (two em dashes) between term and answer, matching existing files.

### Body Content

The body is a **free space for editors** — there is NO fixed section structure. Tailor the content to each idiom rather than forcing a rigid template. Typical sections include:

- `## 字面意思` — Literal meaning
- `## 引申意思` — Extended/figurative meaning
- `## 例子` — Usage examples (blockquote format with `>`)
- `## 來源` — Origin or etymology

If an idiom has interesting stories, historical anecdotes, or cultural context behind it, **include them** — these make the content richer and more engaging. Add, remove, or restructure sections as each idiom demands.

**NO HALLUCINATION (NON-NEGOTIABLE):** Every factual claim must be accurate. If you are unsure about an origin story or historical detail, either omit it or explicitly mark it as uncertain (e.g. 「來源不確定」). Never fabricate citations, sources, or historical events.

### Language Requirements (NON-NEGOTIABLE)

- **ALL prose MUST be 廣東話** (spoken Cantonese), NOT 書面語 (written/literary Chinese)
- Use **traditional Chinese characters** exclusively
- Examples should sound natural as spoken Cantonese
- Match the conversational, clear, educational tone of existing files
- **Use Cantonese vocabulary**, not Mandarin equivalents. Examples:
  - 雪糕 not 冰淇淋
  - 番茄 not 西紅柿
  - 的士 not 出租車
  - 巴士 not 公交車
  - 沖涼 not 洗澡
- Content should sound natural to **native Cantonese speakers** — if a phrase sounds off when read aloud in Cantonese, rewrite it

## Step 4: Verification Subagents

After generating content, spawn **two subagents in parallel** using the Explore agent:

### Subagent 1: Fact-Check

Prompt template:

```
You are a Cantonese language fact-checker. Verify the following idiom content for accuracy. DO NOT write code — this is a research-only task.

IDIOM CONTENT:
<paste the full generated .md content here>

CHECK EACH OF THE FOLLOWING:

1. **Meaning verification:** Is the answer/punchline (「<answer>」) the correct and commonly accepted punchline for this idiom? If there are alternative accepted answers, list them.

2. **Etymology/cultural claims:** Verify any historical or cultural claims made in the content. Flag anything that appears fabricated or cannot be independently verified. If the origin is uncertain or disputed, say so.

3. **Jyutping verification:** Verify each Jyutping romanization against reliable references (粵語審音配詞字庫, words.hk, 漢典). List each character with its expected Jyutping and flag any mismatches:
   - term: <term> → termJyutping: <termJyutping>
   - answer: <answer> → answerJyutping: <answerJyutping>

4. **Source check (MOST IMPORTANT):** Try to find reputable sources confirming this idiom. It is acceptable to find none — **NEVER fabricate citations or invent sources**. If you cannot verify a claim, say so honestly. Note what you found or didn't find.

RETURN FORMAT:
- For each check: PASS, FLAG (uncertain), or FAIL (incorrect)
- For any FLAG or FAIL: explain the issue and suggest corrections
- Overall verdict: APPROVED or NEEDS REVISION (with specific fixes)
```

### Subagent 2: Word Usage Review

Prompt template:

```
You are a traditional Chinese language reviewer specializing in Cantonese. Review the following content for character correctness and natural Cantonese expression. DO NOT write code — this is a research-only task.

IDIOM CONTENT:
<paste the full generated .md content here>

CHECK EACH OF THE FOLLOWING:

1. **Traditional vs simplified Chinese:** Check EVERY character for correctness. The mapping between traditional and simplified is NOT 1-to-1. Common mistakes to watch for:
   - 天後 vs 天后 (后 is correct in traditional for "queen" — 後 means "after")
   - 幹 vs 干 vs 乾 (all valid traditional characters that collapse to 干 in simplified — wrong choice changes meaning)
   - 裏 vs 裡 vs 里 (裏/裡 = inside, 里 = unit of distance or neighbourhood)
   - 著 vs 着 (both exist in traditional with different uses)
   - 佔 vs 占 (佔 = occupy in traditional)
   - 衝 vs 沖 (衝 = rush/charge, 沖 = rinse/flush)
   - 麵 vs 面 (麵 = noodles, 面 = face)
   - 發 vs 髮 (發 = emit/develop, 髮 = hair)
   List every character that may be the wrong traditional variant.

2. **Cantonese vs Mandarin phrasing:** Verify the prose reads as natural 廣東話, not 書面語 or Mandarin-influenced Chinese. Flag any phrases that:
   - Use 書面語 constructions (e.g., 因為...所以..., 雖然...但是..., 如果...就...)
   - Use Mandarin vocabulary where Cantonese equivalents exist (e.g., 東西 instead of 嘢, 什麼 instead of 咩/乜, 冰淇淋 instead of 雪糕, 西紅柿 instead of 番茄, 出租車 instead of 的士)
   - Sound unnatural when read aloud in Cantonese

RETURN FORMAT:
- For each character issue: the character, its location, what's wrong, and the correction
- For each phrasing issue: the phrase, what's wrong, and a natural Cantonese alternative
- Overall verdict: APPROVED or NEEDS REVISION (with specific fixes)
```

## Step 5: Apply Fixes and Finalize

1. Review both subagent reports
2. Apply all corrections flagged as FAIL
3. For items flagged as FLAG (uncertain), use your best judgement — if uncertain, add a note in the content acknowledging the uncertainty
4. If significant changes were needed, re-run the relevant subagent to verify fixes
5. Run `pnpm test` to validate all content passes schema validation

## Common Mistakes

| Mistake                             | Fix                                                                                   |
| ----------------------------------- | ------------------------------------------------------------------------------------- |
| Using 書面語 instead of 廣東話      | Re-read existing files for tone. Use spoken Cantonese constructions                   |
| Wrong traditional character variant | Always verify via word usage review subagent                                          |
| Invalid Jyutping format             | Must match `^[a-z]{1,7}[1-6]( [a-z]{1,7}[1-6])*$`                                     |
| Fabricated etymology                | Mark uncertain origins explicitly; never invent sources                               |
| Duplicate ID                        | Always use `pnpm gen:content` — it handles ID generation and duplicate file detection |
| Missing `──` in title               | Use fullwidth `──` (two em dashes), not `-` or `—`                                    |
